# Diagram Style

Use short node labels and move details out of boxes. ASCII diagrams break when boxes contain too much prose.

This applies to diagrams in chat, Markdown, README files, docs, specs, PR comments, issue comments, terminal output, and code comments.

Prefer:

- One flow direction.
- One primary path.
- Small clusters.
- Edge labels under five words.
- Multiple diagrams for branching systems.
- Rendered output from D2, Graph::Easy, or PlantUML.

Avoid:

- Long bullets inside boxes.
- Deep nesting.
- Crossed arrows.
- Decorative banners.
- Manual alignment for final output.
- Treating README diagrams as the only use case.
